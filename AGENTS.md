# AGENTS.md

## Project goal

Build a new production-style application from scratch.

## Working rules

- Always explain the plan before making major changes.
- Keep the project structure simple and readable.
- Do not add dependencies unless necessary.
- After code changes, run relevant build or test commands.
- When something is unclear, make the safest reasonable assumption and state it.
- Output a summary of files changed after each task.
- Do not use placeholder in any function/method.
- Do not simplify or remove existing code just to hide an unresolved problem.

## Coding preferences

- Prefer maintainable code over clever code.
- Add comments only where they help understanding.
- Keep functions small and focused.

## Delivery standard

- Code should run locally.
- Basic README setup instructions must exist.
- Include a simple verification step for each feature.
